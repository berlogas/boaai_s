Разработай локальную научную платформу "БережинскИИ" для научных учреждений 
на стеке: PaperQA2 v5 + Ollama + FastAPI + Streamlit + Docker.

### АРХИТЕКТУРА И БЕЗОПАСНОСТЬ
- Полная автономность: всё работает локально, данные не покидают инфраструктуру
- Docker Compose: Ollama (LLM), Backend (FastAPI), Frontend (Streamlit)
- PaperQA2 v5 — ядро системы (ретривал + генерация с точными цитатами)
- Поддержка моделей через Ollama: llama3.1:8b/70b, mistral, qwen2.5 и др.
- Эмбеддинги: nomic-embed-text (или аналоги через Ollama)

### СИСТЕМА РОЛЕЙ (RBAC)
Две роли с чётким разграничением:

1. АДМИНИСТРАТОР (полный контроль)
   - Загрузка документов в ГЛОБАЛЬНУЮ БАЗУ ЗНАНИЙ (единственный способ пополнения)
   - Управление пользователями (создание, удаление, назначение ролей)
   - Удаление документов из глобальной базы
   - Переиндексация, аудит активности
   - Просмотр всех сессий

2. ИССЛЕДОВАТЕЛЬ (рабочее пространство)
   - Создание ЛИЧНЫХ СЕССИЙ (изолированных проектов)
   - Загрузка ВРЕМЕННЫХ ДОКУМЕНТОВ в сессию (не влияют на глобальную базу!)
   - Запросы к базе знаний с разными режимами поиска
   - Генерация научных текстов с цитированием

### ПЕРСИСТЕНТНОСТЬ СЕССИЙ (КЛЮЧЕВОЕ ТРЕБОВАНИЕ)
Сессия исследователя:
- НЕ закрывается при выходе из системы
- Сохраняется 90 дней с последней активности (продлевается при каждом входе)
- При повторном входе исследователь возвращается к точке остановки:
  * Все загруженные документы на месте
  * История чата сохранена
  * Проекты статей в том же состоянии
  * Видно, где остановился (последнее действие)
- Можно иметь несколько сессий параллельно (разные статьи/проекты)
- Автосохранение состояния каждые 5 минут (heartbeat)

### РЕЖИМЫ РАБОТЫ ИССЛЕДОВАТЕЛЯ

1. СВОБОДНОЕ ОБЩЕНИЕ (без сессии)
   - Быстрые вопросы системе без входа в рабочую сессию
   - Используется только глобальная база знаний
   - История сохраняется отдельно
   - Примеры: "Расскажи о методах измерения биомассы", "Что такое CTD-зонд?"

2. РАБОТА В СЕССИИ (структурированная работа над статьёй)
   - Полный доступ к рабочему пространству
   - Загрузка личных документов
   - Создание проектов статей
   - RAG fusion: глобальная база + личные материалы

### РАБОЧЕЕ ПРОСТРАНСТВО ИССЛЕДОВАТЕЛЯ (внутри сессии)

КАТЕГОРИИ ДОКУМЕНТОВ:
- project_draft — черновики своей статьи (intro, methods, results, discussion)
- project_data — данные и результаты (CSV, Excel, таблицы)
- temp_literature — статьи коллег, которых нет в общей базе (препринты, свежие публикации)
- notes — собственные заметки и анализы (TXT, MD)
- supplementary — дополнительные материалы (методички, инструкции)

ПРОЕКТЫ СТАТЕЙ:
- Создание структурированных проектов с планом разделов
- Привязка документов к проекту (как reference или data)
- Целевой журнал, статус работы
- Автоматический анализ черновика на соответствие литературе

СВЯЗИ МЕЖДУ ДОКУМЕНТАМИ:
- Связь черновика с данными
- Связь с цитированием (cites, supports, contradicts)
- Пользовательские теги

### RAG FUSION (ОБЪЕДИНЁННЫЙ ПОИСК)

РЕЖИМЫ ПОИСКА:
- hybrid (по умолчанию) — глобальная база + рабочее пространство, приоритет личным документам
- project_focus — приоритет документам активного проекта
- session_only — только загруженные в сессию файлы
- global_only — только проверенная общая база (для свободного общения)

ПРИОРИТЕТЫ:
1. Документы активного проекта (высший приоритет)
2. Собственные данные и черновики (project_data, project_draft)
3. Временная литература сессии
4. Глобальная база знаний

Результат: ответ с пометками источников (📚 общая литература vs 📁 ваши материалы)

### ФУНКЦИИ ГЕНЕРАЦИИ

Для исследователя:
- Генерация обзора литературы с цитированием
- Генерация разделов статьи (Introduction, Methods, Results, Discussion, Conclusions)
- Сравнение своих данных с литературой (ключевой инструмент для Discussion!)
- Генерация отчётов (HELCOM и др. форматы)
- Анализ черновика: каких цитат не хватает, пробелы в аргументации
- Поиск пробелов в литературе: какие источники нужно найти

### ЛИМИТЫ И УПРАВЛЕНИЕ
- Макс. сессий на исследователя: 10
- Макс. документов в сессии: 50
- Лимит хранилища сессии: 500 МБ
- Макс. проектов статей в сессии: 5
- Автоудаление сессий старше 90 дней без активности

### ОГРАНИЧЕНИЯ PAPERQA2 V5 (требуют собственной реализации)

PaperQA2 v5 предоставляет:
- ✅ Парсинг PDF, чанкирование, векторную индексацию
- ✅ Ретривал с реранкингом по релевантности
- ✅ Генерацию ответов с цитатами из контекста
- ✅ Сохранение/загрузку индексов

PaperQA2 v5 НЕ предоставляет (реализуем самостоятельно):
- ❌ Мультипользовательность → изолированные Docs для каждой сессии
- ❌ Приоритизацию источников → ручной reranking по категориям/проектам
- ❌ Объединение нескольких индексов → RAG fusion через litellm.completion()
- ❌ Потокобезопасность → блокировки на уровне сессий
- ❌ Персистентность сессий → собственное хранилище JSON + heartbeat
- ❌ Свободное общение без сессии → отдельный endpoint /quick-query

### АРХИТЕКТУРНЫЕ РЕШЕНИЯ

RAG Fusion (объединение источников):
1. Отдельный query() в глобальный Docs
2. Отдельный query() в сессионный Docs (по категориям)
3. Слияние результатов с приоритизацией
4. Генерация финального ответа через litellm с явным контекстом

Персистентность:
- Сессии хранятся в JSON (users.json, sessions.json)
- Heartbeat каждые 5 минут продлевает срок жизни
- При входе: dashboard с активными сессиями и точкой возврата
- Автовосстановление состояния при resume

Потокобезопасность:
- Lock на уровне сессии при модификации индекса
- Lock на глобальный индекс для админских операций
- Read-only операции (query) без блокировок

### ИНТЕРФЕЙС ИССЛЕДОВАТЕЛЯ

Главная страница (дашборд):
- Быстрый вопрос (без сессии)
- Список активных сессий с превью последней активности
- Кнопка "Продолжить [название]" для последней сессии
- Создание новой сессии
- Индикация срока жизни сессий (предупреждение об истечении)

Внутри сессии:
- Загрузка документов с выбором категории
- Просмотр по категориям (черновики, данные, литература, заметки)
- Управление проектами статей
- Чат с выбором режима поиска
- Визуальная индикация источников (📚 vs 📁)

### СЦЕНАРИЙ ИСПОЛЬЗОВАНИЯ

Исследователь Иванов:

1. Понедельник (первый вход):
   - Зашёл в систему, видит пустой дашборд
   - Задал быстрый вопрос: "Какие методы измерения биомассы существуют?"
   - Создал сессию "Планктон_Баренцево_2024"
   - Загрузил CSV с данными (project_data)
   - Создал проект статьи, привязал данные

2. Вторник (работа в сессии):
   - Вошёл в сессию, всё на месте
   - Загрузил черновик Introduction (project_draft)
   - Спросил: "Проанализируй моё введение"
   - Система нашла недостающие цитаты

3. Среда (выход из системы):
   - Нажал "Приостановить", написал заметку "Нужно найти статью Smith 2023"
   - Вышел из системы

4. Пятница (возврат к работе):
   - Вошёл в систему, видит сессию "Планктон_Баренцево_2024" в статусе "paused"
   - Нажал "Продолжить", видит: "Последнее: анализ введения, заметка: нужна Smith 2023"
   - Загрузил статью Smith (temp_literature)
   - Сгенерировал Discussion сравнивая свои данные с Smith

5. Через 2 месяца:
   - Вошёл, сессия активна (автопродление при работе)
   - Дописал статью, экспортировал в Word
   - Закрыл сессию (архивация)

6. Через 100 дней без входа:
   - Сессия автоматически архивирована (только чтение)
   - Уведомление: "Ваша сессия архивирована. Скачайте материалы или обратитесь к администратору для восстановления."
